This test asks whether the following execution is architecturally Allowed:

\begin{itemize}
  \item The first Store instruction in program order on P0, i.e. STR W0,[X1], generates an Explicit Memory Write Effect of Location x with value 1;
  \item The second Store instruction in program order on P0, i.e. STR W2,[X3], generates an Explicit Memory Write Effect of Location y with value 1;
  \item The first Store instruction in program order on P1, i.e. STR W0,[X3], generates an Explicit Memory Write Effect of Location y with value 2;
  \item The second Store instruction in program order on P1, i.e. STR W2,[X4], generates an Explicit Memory Write Effect of Location z with value 1;
  \item The first Load instruction in program order on P2, i.e. LDR W5,[X4], generates an Explicit Memory Read Effect of Location z;
  \item The second Load instruction in program order on P2, i.e. LDR W6,[X1], generates an Explicit Memory Read Effect of Location x;
  \item The value of y is 2 in the end, which means that the Explicit Memory Write Effect of Location y on P0 generated by STR W2,[X3] is Coherence-before the Explicit Memory Write Effect of Location y on P1 generated by STR W0,[X3];
  \item The Register X5 on P2 holds the value 1 in the end, which means that the Explicit Memory Read Effect of Location z on P2 generated by LDR W5,[X4] Reads-from the Explicit Memory Write Effect of Location z on P1 generated by STR W2,[X4];
  \item The Register X6 on P2 holds the value 0 in the end, which means that the Explicit Memory Read Effect of Location x on P2 generated by LDR W6,[X1] Reads-from the initial state, and therefore is Coherence-before the Explicit Memory Write Effect of Location x on P0 generated by STR W0,[X1].
\end{itemize}
